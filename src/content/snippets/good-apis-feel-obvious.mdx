---
title: "Good APIs Feel Obvious"
description: "When an API is well-designed, it feels like the only way to solve the problem."
date: "2026-06-15"
tags: ["api-design"]
status: "published"
---

Good API design is invisible.

You use it and think: "Of course it works this way. How else would you do it?"

When you have to read documentation to understand a simple operation, the API failed. When you keep getting method names wrong because they don't match your mental model, the API failed.

The best APIs feel inevitable. They align with how people naturally think about the problem.

This is why designing APIs is hard. You have to understand not just what the API does, but how people expect it to work.
